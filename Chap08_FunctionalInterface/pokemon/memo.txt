Java の インタフェース（interface） は、
「クラスにやってほしい行動（メソッドの形）」だけを決める設計図 です。

🎯 インタフェースとは一言でいうと？

中身のないメソッドだけを宣言して、
「このクラスはこういう行動ができます！」と約束させる仕組み

📌 例：ポケモンで説明

ポケモンには「鳴く」という動作がありますが、
ポケモンごとに鳴き声は違いますよね？

ピカチュウ →「ピカチュウ！」

ゼニガメ →「ゼニゼニ！」

イーブイ →「イーブイ！」

これを Java で表すと：

🔹 インタフェース（鳴くという行動の定義）
interface Cry {
    void cry();
}


これは 「cry() という行動を持ちなさい」
と決めているだけ。
中身は書かない。

🔹 ポケモンごとに実装（行動の中身）
class Pikachu implements Cry {
    public void cry() {
        System.out.println("ピカチュウ！");
    }
}

class Zenigame implements Cry {
    public void cry() {
        System.out.println("ゼニゼニ！");
    }
}


ここでは 中身（鳴き声）を各クラスが書く。

🧠 なぜインタフェースを使うの？
① 共通の行動をまとめられる

Cry を実装したポケモンは全部「cry() を持っている」ことが保証される。

Cry p = new Pikachu();
Cry z = new Zenigame();

p.cry();  // ピカチュウ！
z.cry();  // ゼニゼニ！
